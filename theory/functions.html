Functions

We already talked about functions briefly, but let's now take a deeper look. The template for a new function is like this: () { }

The name of the function is in camelCase, just like variables.

Arguments

Functions can take arguments. This can be used to change the exact way the function works, or to pass through data that the function uses to do things. Here is a simple example: main() { greet("Bert"); greet("Ernie"); } greet(String friend) { print("A warm welcome to..."); print(friend); } Here, the function greet takes 1 argument of type String and we give it the name friend. Note how it looks almost exactly like creating a variable normally, except we don't give it a value. This is because the value is assigned the moment the function is called, like in this example where we pass the value "Bert" and then "Ernie".

The first time the function is called, the argument "Bert" is passed between the brackets. This is a String, so this doesn't give any errors. greet is then executed with the friend variable set to "Bert". It prints A warm welcome to... followed by Bert. After that we reach the end of the greet function, so we go back to main.

Immediately after, greet is called again, but this time the friend argument receives the value "Ernie". So when executing, it now prints A warm welcome to... Ernie instead.

Functions can take any amount of arguments, and they don't have to be of the same type. If you really need to, you can make very long functions, like this monstrosity from the Flutter framework. createLabelWrap(String label, int itemCount, Button childButton, Padding padding, Margins margins, bool hasBorder, bool scrollable, bool isFilled, Color fillColor) { // I omitted the actual code, it's quite complicated } If you have more than one argument, don't forget to separate them with commas!

Return Types

That's great and all, but what if we want to make a function do something, and get back the result? That is done through return types. Let's consider this function that adds 2 numbers: main() { add(2, 3); // how do we retrieve the result here? } add(int a, int b) { int sum = a + b; } We need to do 2 things. First, we of course need to specify the data type of the variable we want to return. In this case, we want to return an int. This is specified by putting the data type (in this context, we call it the 'return type') in front of the function name: int add(int a, int b) { int sum = a + b; } If you try this in DartPad, you will notice it's giving us a warning. That's because we promised to return an int, but we haven't returned anything yet! To return the calculated sum, we have to use the return keyword, followed by an expression that resolves to a value of our return type. We can simply use the sum variable we created. int add(int a, int b) { int sum = a + b; return sum; } Since return accepts expressions, we can even write this shorter, since a + b resolves to an int: int shorterAdd(int a, int b) { return a + b; }

If you don't specify any return type, there is a default data type called void. This type means that the function returns no value at all. So far none of the examples had a return type, so Dart automatically assumes we don't return anything and fills in the void behind the scenes. For example, you can write the main function like this as well: void main() { }
